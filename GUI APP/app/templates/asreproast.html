About AS-REP Roasting

AS-REP Roasting targets Active Directory accounts that have the "Do not require Kerberos preauthentication" flag enabled. When preauthentication is disabled, anyone can send an AS-REQ to the KDC on behalf of that user without proving knowledge of their password. The KDC responds with an AS-REP containing an encrypted portion derived from the user's password hash. An attacker can capture this response and crack it offline to recover the plaintext password. This technique requires only network access to the domain controller — no domain credentials are needed if the target usernames are already known. It is especially dangerous when combined with weak or predictable passwords.

Recommendation

Ensure Kerberos preauthentication is required for all user accounts — review and remove the "Do not require Kerberos preauthentication" flag wherever it has been set. Enforce strong, complex passwords and use Group Managed Service Accounts (gMSAs) where applicable. Regularly audit accounts with this flag using LDAP queries and monitor for anomalous AS-REQ traffic to detect potential abuse early.
